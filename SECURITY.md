# 安全策略

harness-cook 是一个 Agent 治理工具——安全是它的核心场景之一。我们同样严肃对待本项目自身的安全。

## 支持版本

本项目处于 Alpha 阶段，仅对最新主分支与最近发布版本提供安全修复。

| 版本 | 支持状态 |
|------|----------|
| 0.1.x | ✅ 当前支持 |

## 报告漏洞

**请不要通过公开 issue 报告安全漏洞。**

请通过以下任一渠道私密报告：

1. **优先**：使用 GitHub Security Advisories（仓库 → Security → Report a vulnerability），可私下与维护者协作。
2. **或邮件**：发送至 `security@harness-cook.org`（请替换为维护者实际安全邮箱），邮件主题以 `[SECURITY] harness-cook:` 开头。

报告中请尽量包含：
- 受影响版本（git commit 或 tag）
- 漏洞类型（如 RCE、注入、敏感信息泄露、治理绕过）
- 复现步骤与最小化 PoC
- 影响范围与建议修复方向（如有）

## 响应流程

1. 收到报告后 3 个工作日内确认接收。
2. 维护者评估漏洞等级与影响范围，与报告者协作确认。
3. 修复在非公开分支完成后，协调发布与公开披露。
4. 在修复发布后致谢报告者（除非报告者要求匿名）。

我们请求报告者在修复发布前不要公开披露漏洞细节，以便为受影响用户提供修复窗口。

## 治理绕过类漏洞

由于本工具定位为 Agent 行为约束，以下情况属于高优先级漏洞：
- Profile / Gate 配置可被 Agent 绕过而未触发审计
- Hook 注入路径在特定平台失效且未降级
- 审计链可被篡改或伪造
- PII / 敏感信息在应脱敏的路径未脱敏

报告此类漏洞时请说明触发的 Agent 平台与 Profile 配置。
